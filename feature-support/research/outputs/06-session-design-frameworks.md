# Phase 6: Engagement and Session Design Frameworks

## Core Finding

Theoryâ†’Methodâ†’Application provides the right macro arc for session design, but it is not sufficient as a complete session-level framework. It lacks four elements that the research consistently identifies as necessary: prior knowledge activation, formative assessment checkpoints, structured reflection, and explicit transfer design. The curriculum builder needs a session template that wraps TMA inside these bookend elements â€” and it needs multiple templates, because different session types (knowledge acquisition, inquiry/exploration, problem-solving) require structurally different approaches.

## What the Research Says About Session Design

### 1. Learning Cycles Are Widely Adopted, Rarely Fully Implemented

The most consistent finding across learning cycle research is that practitioners systematically skip the phases that matter most. A 2025 systematic review of 30 studies using Kolb's experiential learning cycle found that only 2 (6.7%) fully implemented all four phases. Morris (2020) confirmed the pattern: Reflective Observation and Abstract Conceptualization are the most commonly truncated phases. Practitioners favor doing over thinking. [Source: experiential-learning-and-transformative-learning.md]

The same pattern appears in the 5E model. Teachers tend to emphasize Engage and Explain while cutting Elaborate and Evaluate â€” the phases that require students to apply, extend, and self-assess. The phases that produce transfer and consolidation are the phases that get dropped. [Source: experiential-learning-and-transformative-learning.md]

This finding has a direct implication for the curriculum builder: the phases that get skipped in practice are the phases the tool must make hardest to skip. Schema enforcement (Phase 5's primary constraint mechanism) should apply most rigorously to reflection, formative assessment, and transfer design â€” because these are the elements that human instructors naturally omit. [Source: cognitive-load-session-pacing-tma.md]

### 2. The 5E Model Has the Strongest Quantitative Evidence

The Polanin et al. (2024) meta-analysis â€” 61 RCTs, 156 effect sizes, 70% meeting WWC standards â€” found the 5E model produced a large positive effect on science outcomes (g = 0.82). The 7E extension (which adds explicit prior-knowledge elicitation and far transfer) showed even larger effects (g = 1.23) compared to the 3E model (g = 0.42). The message is clear: the more complete the learning cycle, the stronger the effect. Truncating phases reduces impact proportionally. [Source: session-design-templates-evidence.md]

Direct/explicit instruction also shows strong effects (d = 0.59 in Hattie's meta-analyses), and the EDI research produced a finding that should be this project's motto: "Effective instruction requires both well-designed lessons and well-taught lessons â€” you can't compensate for poor design with great delivery." This is the foundational justification for a design tool that enforces good session structure. [Source: session-design-templates-evidence.md]

### 3. TMA Needs Bookends

Theoryâ†’Methodâ†’Application maps cleanly to established instructional sequences â€” GagnĂ©'s events 4-9, the Gradual Release of Responsibility (I Do / We Do / You Do), Merrill's Demonstration-Application cycle. It is a valid and well-grounded session arc. But it is missing elements that every major framework includes. [Source: cognitive-load-session-pacing-tma.md]

What TMA provides: a scaffolded progression from conceptual understanding through structured process to open-ended application. This is sound.

What TMA omits:

**Prior Knowledge Activation** â€” GagnĂ© Event 3, Merrill's Activation principle, the 5E's Engage phase, and Rosenshine's Principle 1 all begin by connecting new material to what learners already know. TMA's Theory phase starts with the theory, not with the learner. For novice audiences, this works. For experienced adults who bring substantial prior knowledge (the doctrine's target audience), skipping activation means missing the opportunity to create cognitive hooks and â€” more critically â€” to surface misconceptions that will interfere with learning.

**Formative Assessment Checkpoints** â€” GagnĂ© Events 6-7, Rosenshine's Principles 3 and 6, and the 5E's Evaluate-throughout design all require checking understanding DURING the session. TMA has no explicit check between Theory and Method, or between Method and Application. Cognitive load research shows that proceeding to application when conceptual understanding is incomplete produces frustration, not learning.

**Structured Reflection** â€” Kolb's Reflective Observation, Merrill's Integration principle, and Mezirow's critical assessment of assumptions all require a pause to consolidate learning. TMA's Application phase is about doing; there is no phase for stepping back and asking "what did I learn, what changed, what do I now believe that I didn't before?"

**Transfer Design** â€” GagnĂ© Event 9, Merrill's Integration principle, and the 7E's Extend phase all explicitly address how learning transfers beyond the session. TMA's Application is typically context-bound â€” "apply the method to this problem." Transfer design asks the harder question: "how does this apply to the different problems you'll face next week?"

The recommended session template wraps TMA in these elements:

```
ACTIVATE â†’ THEORY â†’ CHECK â†’ METHOD â†’ PRACTICE â†’ APPLICATION â†’ REFLECT â†’ TRANSFER
```

This is not a rigid sequence â€” it's a minimum set of required elements, some of which can be reordered or compressed depending on session type and audience expertise. [Source: cognitive-load-session-pacing-tma.md]

### 4. The "Aha Moment" Problem â€” From Marketing to Mechanism

The old template's Step 7 treated "aha moments" as marketing copy to brainstorm â€” "Create 2-3 'Aha' ideas that represent key concepts, critical beliefs, essential skills." Mezirow's transformative learning theory provides a far more rigorous framework. [Source: experiential-learning-and-transformative-learning.md]

Perspective transformation is not something a curriculum designer can script. It is a cognitive event triggered by specific conditions: a disorienting dilemma (evidence or experience that cannot be assimilated into existing mental models), followed by critical reflection on assumptions, rational discourse with others, and provisional action in new roles. The curriculum can create the conditions for transformation â€” present disconfirming case studies, structure peer dialogue, require assumption-testing exercises â€” but it cannot guarantee the aha.

The curriculum builder should replace "brainstorm aha ideas" with a structurally rigorous alternative: **belief-challenging encounters**. For each module, the builder should generate:

1. **A disorienting dilemma** â€” a case study, data point, or scenario that challenges a common assumption the target audience holds
2. **A reflection prompt** â€” "What did you assume before this module? What do you assume now? What changed?"
3. **A peer dialogue structure** â€” a discussion format that surfaces and tests changed beliefs

This moves "aha moments" from marketing content (something the curriculum promises) to pedagogical mechanism (something the curriculum creates conditions for). [Source: experiential-learning-and-transformative-learning.md]

### 5. Cognitive Load and Session Pacing

The "students can only pay attention for 10-15 minutes" claim is poorly supported. Wilson & Korn (2007) reviewed the evidence and found that attention lapses occur much more frequently and irregularly â€” at 30 seconds, 4-5 minutes, 7-9 minutes. The relevant finding is not that there is a fixed attention window, but that activity switching reliably resets engagement. A change in modality (from listening to discussing, from individual to group, from conceptual to applied) is the most effective attention management tool. [Source: cognitive-load-session-pacing-tma.md]

Cognitive load theory (Sweller, 1988) provides more actionable guidance for session design: present new material in small chunks of 2-4 interacting elements; use worked examples before independent practice for novices; reduce split attention by integrating related information; avoid redundancy (don't read slides aloud). The expertise reversal effect (Kalyuga & Sweller, 2003) complicates this: the instructional strategies that help novices (detailed guidance, worked examples) actively harm experts (create redundancy load). This reinforces the Phase 2 finding that learner expertise level must be captured in the intake and used to adapt session design. [Source: cognitive-load-session-pacing-tma.md]

Interleaving â€” mixing different problem types within a practice activity rather than blocking same types together â€” produces approximately 30% improvement in learning within a single session, with effects as large as 125% on delayed tests in physics education (Nature, 2021). This is a session-level practice design principle the builder should implement by default. [Source: cognitive-load-session-pacing-tma.md]

### 6. Multiple Session Templates, Not One

The cross-framework analysis across GagnĂ©, Merrill, Rosenshine, Hunter, 5E, and WIPPEA reveals a universal session arc with four macro phases: Open (activate prior knowledge, state purpose) â†’ Present (introduce new material with demonstrations) â†’ Practice (guided then independent, with feedback) â†’ Close (assess, consolidate, transfer). Every major framework contains these four elements, arranged in slightly different configurations. [Source: session-design-templates-evidence.md]

But the research also shows that different session types need different emphasis patterns:

| Session Type | Best Framework | Why |
|---|---|---|
| Structured knowledge/skill transfer | GagnĂ©'s 9 Events | Most prescriptive event-by-event sequence; strong for declarative and procedural knowledge |
| Inquiry and exploration | 5E/7E Model | Designed for discovery; students construct understanding through exploration before explanation |
| Problem-centered professional development | Merrill's First Principles | Starts with real-world tasks; emphasizes integration into professional context |
| Adult professional learning | WIPPEA | Warm-up addresses adult motivational needs; Application as final phase emphasizes transfer |

The builder should offer framework-appropriate templates rather than forcing all sessions through a single structure. The intake should capture not just WHAT is being taught but WHAT KIND OF LEARNING the session targets â€” and select the appropriate template accordingly. [Source: session-design-templates-evidence.md]

### 7. The Over-Prescription Warning

Both Rosenshine and Madeline Hunter's legacies contain a critical caution. Rosenshine explicitly warned against converting his principles into "mandatory lesson structures, observed through compliance checklists." Hunter's flexible instructional elements were compiled by others into a rigid "Seven-Step Lesson Plan" that she never intended. In both cases, the loss of flexibility reduced effectiveness. [Source: session-design-templates-evidence.md]

This is directly relevant to the curriculum builder. The tool must balance structure (enforcing the elements that research shows matter) with flexibility (not prescribing rigid sequences that may not fit the content or audience). The resolution: **required elements with flexible ordering**. The builder should enforce that every session includes activation, practice, feedback, reflection, and transfer â€” but should not dictate the exact order or time allocation for each.

## What This Means for the Engine

### Session Schema: Required Fields

Every generated session must include these fields (non-negotiable, schema-enforced):

1. `prior_knowledge_activation` â€” How existing knowledge is surfaced and connected
2. `learning_objectives` â€” What learners will be able to do (inherited from Phase 5 pipeline)
3. `content_chunks` â€” New material broken into segments of manageable cognitive load
4. `formative_check` â€” At least one understanding check before major practice
5. `guided_practice` â€” Structured practice with support (before independent practice)
6. `independent_practice` â€” Application with reduced scaffolding
7. `reflection_prompt` â€” What was learned, what changed, what assumptions were tested
8. `transfer_connection` â€” How this applies beyond the session context

### Session Schema: Recommended Fields

These should be generated when appropriate but are not validation failures if absent:

- `belief_challenging_encounter` â€” Disorienting dilemma per Mezirow
- `peer_dialogue_structure` â€” Discussion format for testing changed beliefs
- `interleaved_practice_design` â€” Mixed problem types rather than blocked
- `activity_type_switches` â€” Modality changes at ~15-minute intervals

### Template Selection Logic

The builder should select session template based on session type:

```
IF session_type == "knowledge_transfer" â†’ GagnĂ©-derived template (9 events)
IF session_type == "inquiry_exploration" â†’ 5E/7E-derived template
IF session_type == "problem_solving" â†’ Merrill-derived template (task-centered)
IF session_type == "professional_development" â†’ WIPPEA-derived template
DEFAULT â†’ Universal Arc (Open-Present-Practice-Close) with TMA overlay
```

### TMA Integration

TMA maps to the content delivery portion of any template:

- **Theory** â†’ Content presentation phase (GagnĂ© Events 4-5, 5E Explain, Merrill Demonstration)
- **Method** â†’ Guided practice phase (GagnĂ© Event 6, 5E Elaborate, Merrill Application)
- **Application** â†’ Independent practice + transfer (GagnĂ© Events 8-9, 5E Evaluate, Merrill Integration)

TMA is the CONTENT arc within the session. The session template is the COMPLETE arc that includes activation before and reflection/transfer after.
