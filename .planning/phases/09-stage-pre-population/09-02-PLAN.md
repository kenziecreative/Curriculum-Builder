In `intake.md`, find the Audit Mode **Step 6: Write Output Files** section. After step **4. Write `curriculum-gap-report.md`** (the gap report write) and before step **5. Update STATE.md silently**, insert a new numbered step:

**Step 5 (new): Pre-populate stage files from gap report**

The new step must:

**A. Gap report guard:** Check if `curriculum-gap-report.md` was just written (it always will be in audit mode, but add an explicit note: "If curriculum-gap-report.md is absent — because this is clean intake, not audit mode — skip this entire step and proceed to STATE.md update"). This handles the edge case from RESEARCH.md Pitfall 5.

**B. Read the gap report Summary table:** Parse which stages are Exists, Shallow, or Missing. Stages 2–5 are eligible for pre-population (Stages 6–8 are never pre-populated — transfer design, metaskills, and marketing require generation, not extraction).

**C. For each eligible stage (2, 3, 4, 5) — if status is Exists or Shallow:**
- Load the corresponding schema file as generation context:
  - Stage 2: `.claude/reference/schemas/stage-02-outcomes.md`
  - Stage 3: `.claude/reference/schemas/stage-03-assessments.md`
  - Stage 4: `.claude/reference/schemas/stage-04-modules.md`
  - Stage 5: `.claude/reference/schemas/stage-05-sessions.md`
- Extract content from source documents already in context
- Write extracted content faithfully to the stage directory — do not auto-correct violations during this write
- For any field that fails schema, add an inline marker: `# NEEDS: [plain language description of what's missing] — /curriculum:[command] will fix this`
- Track what was written and what violations were marked for the summary table

**D. Sessions pre-population constraint (Stage 5):** Write `session-manifest.md` only — session names, mapped outcome IDs, session template placeholder with NEEDS: marker, pre-work if found. Hard limit: no facilitator guide content, no participant materials, no slide outlines.

**E. Write all files simultaneously in one pass.** Never write Stage 2, pause, then Stage 3. All writes happen together. (Established one-pass rule from Phase 5.)

**F. Update STATE.md:** For each stage that received pre-populated files, set that stage's Status to `pre-populated`. (Stages marked Missing remain `not-started`.) This STATE.md update goes here, before the existing step 5 STATE.md write — merge both writes into a single simultaneous STATE.md write.

**G. Display post-intake summary table** (immediately after all writes, before the forward-looking message):

```
| Stage | What was written | Issues found |
|-------|-----------------|--------------|
| 2: Learning Outcomes | [files written, e.g., "learning-objectives.md, enduring-understandings.md, essential-questions.md"] | [plain-language count of NEEDS: markers, e.g., "2 verb violations, 1 missing transfer context" — or "None"] |
| 3: Assessment Design | [files] | [issues] |
| 4: Module Structure | [files] | [issues] |
| 5: Session Content | session-manifest.md (structure only) | [issues] |
| 6: Metaskill Mapping | — (Missing — will generate fresh) | — |
```

Include only stages 2–5 in the table. Stages that were Missing show "— (Missing — will generate fresh)". Stages 6–8 are not shown.

Then update the final forward-looking message in step 6 to:

**Renumber:** The existing step **5. Update STATE.md silently** becomes step **6**, and the existing step **6. End with a forward-looking message** becomes step **7**. Both are now merged/updated as described above.

PREREQ_STATUS=$(grep -A1 "| $PREREQ_NUM " "$STATE_FILE" | grep -oE 'not-started|in-progress|pre-populated|complete' | head -1)

REASON="Stage $STAGE_NUM ($STAGE_NAME) requires Stage $PREREQ_NUM ($PREREQ_NAME) to be complete. Stage $PREREQ_NUM is currently ${PREREQ_STATUS:-not-started}. Run $PREREQ_CMD to finish it first."
