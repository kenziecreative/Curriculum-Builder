Create the `.claude/reference/schemas/` directory if it does not exist.

Each schema file follows a consistent internal structure:
1. Header with stage number, name, command reference, depends_on (which prior stages must be complete), and output directory
2. Required Fields section — each field has: name, type (enum with explicit values, string, array, object), required (yes/no), description, and constraints
3. Enumerated Values section — every field that has a constrained value set lists the exact enum values
4. Duration Scaling section — how this stage adapts for short (90 min) vs. medium (half-day to 2 days) vs. long (multi-week/semester) programs
5. Validation Rules section — machine-checkable conditions that must be true for stage completion

**stage-01-intake.md:**
- depends_on: none (first stage)
- output_dir: `00-project-brief/`
- Required fields from research brief: program_topic, target_audience (with prior_knowledge as "can do X, cannot yet do Y"), transfer_context, success_criteria, duration_and_format (with enums for delivery_mode: [cohort-based, self-paced, hybrid], modality: [in-person, virtual, blended], contact_hours: number), self_direction_level (enum: [Stage 1 - Dependent, Stage 2 - Interested, Stage 3 - Involved, Stage 4 - Self-Directed] per Grow's model), skill_type (enum: [open, closed]), cultural_orientation (enum: [individualist, collectivist, balanced])
- Duration scaling: short programs require fewer intake questions (skip cultural_orientation if under 2 hours); long programs add cohort_size and prerequisite_programs fields
- Validation rules: all required fields populated, target_audience uses behavioral description not self-report labels

**stage-02-outcomes.md:**
- depends_on: stage-01-intake
- output_dir: `01-outcomes/`
- Output files: enduring-understandings.md, essential-questions.md, learning-objectives.md
- Required fields per objective: outcome_statement, bloom_level (enum: [Remember, Understand, Apply, Analyze, Evaluate, Create]), prerequisite_knowledge, transfer_context
- Bloom's distribution rule: program must span at least 4 taxonomy levels
- Expertise-adaptive rule: novice audiences (Grow Stage 1-2) start at Remember/Understand, build to Apply/Analyze; experienced audiences (Grow Stage 3-4) can start at Apply/Analyze, push to Evaluate/Create
- Outcome hierarchy: program_outcomes > module_outcomes > session_outcomes (each level inherits from above)
- Duration scaling: 90-min programs may have 2-3 objectives total; semester programs may have 15-25 across modules

**stage-03-assessments.md:**
- depends_on: stage-02-outcomes
- output_dir: `02-assessments/`
- Output files: assessment-map.md, formative-assessments.md, summative-assessments.md
- Required fields per assessment: assessment_name, paired_objective (reference to specific objective ID), assessment_type (enum: [formative, summative]), bloom_level (same enum as outcomes), format (enum: [performance-based, procedural, reflective, portfolio, case-analysis, simulation, demonstration, written, oral, peer-review])
- Alignment constraint: assessment bloom_level must be >= paired_objective bloom_level
- Skill-type constraint: open-skill curricula require performance-based assessments; closed-skill curricula require procedural assessments
- Formative requirement: at least one formative assessment per module
- Summative requirement: at least one summative assessment at program level
- Duration scaling: 90-min programs use formative checks only (no summative); semester programs require full assessment matrix

Do NOT include validation-agent logic or runtime enforcement code. These are reference spec files that generation commands will include as context. The instruction in each schema should be: "Generate output matching this schema. All required fields must be present. Enum fields must use exact values listed."

**stage-04-modules.md:**
- depends_on: stage-01-intake, stage-02-outcomes, stage-03-assessments
- output_dir: `03-modules/`
- Output files: sequence-rationale.md, per-module directories
- Required fields per module: module_name, learning_objectives (inherited references from Stage 2), prerequisite_modules (array of module IDs, can be empty for first module), content_chunks (array, sized for cognitive load: 2-4 interacting elements for novices per research), social_learning (object with 4 required sub-fields: activity_type [enum: individual, paired, small-group, full-cohort], interdependence_structure, accountability_mechanism, group_processing_prompt), primary_metaskill (enum: [Exploring, Creating, Innovating, Adapting, Imagining, Communicating] — note: pending decision on 7th metaskill, use these 6 for now), metaskill_activation_activity (a specific thinking routine name, NOT a generic label), belief_challenging_encounter (required: at least one per module), modality_switches (required: at least one activity type switch per session)
- Module sequencing constraint: prerequisite_modules must form a valid DAG (no circular dependencies)
- Duration scaling: 90-min programs produce 1 module; half-day produces 2-3 modules; semester produces 8-15 modules

**stage-05-sessions.md:**
- depends_on: stage-04-modules
- output_dir: `04-sessions/` (per user decision, separate top-level dir)
- Required 8 TMA arc fields per session (these are the most critical schema fields in the entire system — they enforce pedagogical completeness):
  1. prior_knowledge_activation (ACTIVATE phase)
  2. learning_objectives (THEORY phase)
  3. content_chunks (THEORY phase, with cognitive load sizing)
  4. formative_check (CHECK phase)
  5. guided_practice (METHOD phase)
  6. independent_practice (PRACTICE phase)
  7. reflection_prompt (REFLECT phase — must be specific, not generic "what did you learn?")
  8. transfer_connection (TRANSFER phase — must reference real-work application from intake transfer_context)
- TMA content arc mapping: ACTIVATE > THEORY > CHECK > METHOD > PRACTICE > APPLICATION > REFLECT > TRANSFER
- Session template selection enum: [gagne, 5e_7e, merrill, wippea, universal_tma_arc]
- Template selection guidance: knowledge transfer -> gagne, inquiry/exploration -> 5e_7e, problem-solving -> merrill, professional development -> wippea, default -> universal_tma_arc
- DCR integration fields: deconstruct_element, compare_element, rebuild_element, validate_element (validate required for wicked-domain curricula)
- DCR scaffolding rule: for novice audiences (Grow Stage 1-2), require close comparisons first, then distant; explicitly paired cases; structural cues
- Additional required fields: facilitator_notes (timing cues, common stumbling points, transition guidance), participant_materials (pre-work, handouts, activity worksheets), slide_framework_outline (structural, not visual)
- Duration scaling: 90-min programs have 1-2 sessions; each session under 90 min. Semester programs have multiple sessions per module, each 60-90 min.

Maintain consistent formatting with Task 1 schemas: same header structure, same field specification format, same section ordering.
