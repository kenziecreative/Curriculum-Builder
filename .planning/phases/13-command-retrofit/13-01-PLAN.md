Command file anatomy (from existing commands):
```
---
description: [brief description]
---

**Output Formatting:** ...

# /curriculum:[name]

## Persona
[voice reference + tone description]

## Prerequisites
[what must exist]

## [Generation sections]
...
## [Write section]
...
## Conversation Output
...
```

Warm handoff pattern (from curriculum-voice.md Signature Moves — Named-handoff close):
- Two sentences: first names what was built, second names the next command explicitly
- Example: "Your learning outcomes are written and saved. Run `/curriculum:assessments` to design the assessments."
- No filler, no "I", no insider vocabulary

Context-clear nudge pattern (new in Phase 13):
- One plain-language sentence
- Example: "Your work is saved — clear context before the next step."
- Must not use: "context window", "tokens", "session", instructional jargon
- IMPORTANT: Do NOT add context-clear nudge to metaskills.md or transfer.md — these auto-trigger the next command; the user cannot clear context mid-chain

PRES-01 — Hiding enforcement steps:
- Any numbered sequence in user-facing Conversation Output sections that reads as "Step N:", "Running:", "Checking:", "Validating:", or similar constraint enforcement process language must be replaced with result-only language
- Pattern to scan for: lines starting with "Step [0-9]", lines matching "Running [a-z]", "Checking [a-z]", "Validating [a-z]" in Conversation Output sections
- Replacement: state the result plainly ("Your outcomes include three knowledge objectives and two skill objectives.") without surfacing the enforcement process that produced the result
- Exception: brief plain-language callouts for user attention (e.g., "One objective needs revision.") are acceptable — they report a result, not a step

QUAL-09 — Writing for Clarity:
- All generation sections in command files must include a Writing for Clarity instruction
- Required instruction text (add to each command's generation section): "Write in kernel sentences — one idea per sentence, subject before verb, active voice. No warm-up openers ('In this section we will...', 'Now that we have...', 'Let us explore...'). Start every paragraph with the conclusion, then support it."
- This applies to generation sections that produce user-visible text: Conversation Output instructions, written file format instructions, and any section labeled Generate or Draft

ASCII formatting for outcomes.md (PRES-03 — backlog spec):
```
╔═══════════════════════════════════╗
║  PROGRAM OUTCOMES                 ║
╠═══════════════════════════════════╣
║  [outcome text]                   ║
╚═══════════════════════════════════╝
```
Tree hierarchy for learning objectives (indented with └─ connectors).

Assessments plain-language summary (PRES-04):
Replace the Assessment Alignment Map table (which has Outcome ID / Bloom Match columns) with:
"[N] assessments designed. Learners will [do X, do Y, do Z]. Together they cover [topics]."
