## slide-outline.md

CURRENT FORMAT (replace entirely):
```
# Slide Framework: [session_name]
**Session:** [M-N-S-N] | **Template:** [session_template]

| Section | TMA Phase | Slides | Purpose |
|---------|-----------|--------|---------|
[rows...]
```
Problems: TMA Phase column (QUAL-07), session_template label (PRES-02), no production direction (QUAL-01).

REQUIRED FORMAT:
```
# [session_name] — Slide Outline

## Opening
**Slide 1 — [slide title]**
- **On screen:** [what appears on the slide — title, key visual, bullets if any]
- **Why it matters:** [pedagogical intent — what this slide is doing for the learner]
- **Facilitator:** [what to say or do while this slide is showing]

**Slide 2 — [slide title]**
[same three-field block]

## Core Content
[slides numbered sequentially, continuing from Opening]

## Practice
[...]

## Close
[...]
```
Notes: section headings are plain (Opening / Core Content / Practice / Close are the model; exact labels are Claude's discretion as long as no TMA vocabulary appears). Slides numbered sequentially across all sections. Timing per section is optional (Claude's discretion).

## facilitator-guide.md

CURRENT FORMAT header (strip these fields):
```
**Template:** [session_template]
**TMA Phases:** ACTIVATE > THEORY > CHECK > METHOD > PRACTICE > REFLECT > TRANSFER
```

CURRENT "Common Stumbling Points" format (upgrade):
```
1. [stumbling point] — [facilitator response]
```

REQUIRED "Common Stumbling Points" format:
```
**Watch for:** [observable learner behavior]
**What it means:** [what that behavior signals about learner state]
**Your move:** [specific facilitation action to take]
```
Each stumbling point becomes one Watch for / What it means / Your move block. The heading "Common Stumbling Points" can stay or be renamed (Claude's discretion) — the content structure must change.

## session.md

CURRENT section headers (remove TMA labels):
```
## ACTIVATE: [activity title]
## THEORY: [topic]
## CHECK: [title]
## METHOD: [title]
## PRACTICE: [title]
## REFLECT: [title]
## TRANSFER: [title]
## DCR: [title]
```

REQUIRED: Use plain descriptive headers that describe the activity content, not the TMA phase:
```
## Opening: [activity title]
## [Content topic — no arc label]
## [etc.]
```
The actual content sections should be named by what they contain or do for learners — not by where they sit in the TMA arc.

CURRENT learning objectives listing (QUAL-05 fix):
```
## THEORY: Learning Objectives
- O-1, O-2, O-3
```

REQUIRED:
```
## Learning Objectives
- O-1: [full text of objective 1]
- O-2: [full text of objective 2]
- O-3: [full text of objective 3]
```
The agent must look up the full text from the learning-objectives.md file and include it. IDs alone are not sufficient.

## All session output files — pre-write cleanup:
Before writing any of the four output files (session.md, slide-outline.md, facilitator-guide.md, participant-materials.md), the agent must:
1. Strip all HTML calculation comments (`<!-- ... -->`) from the content
2. Strip all working notes sections
3. Check for unresolved NEEDS: markers — if any exist, surface them to the user rather than writing the file with markers intact

The strip and check instructions must appear as explicit steps in the write sequence, not as general instructions.
