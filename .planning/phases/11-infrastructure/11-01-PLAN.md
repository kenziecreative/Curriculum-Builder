    Create `scripts/release.sh` with the following behavior:
    1. Reads the VERSION file from the repo root (sibling of the scripts/ directory)

    3. Checks for uncommitted changes — exits with an error if any exist (error handling is at discretion per CONTEXT.md, and this is the recommended approach per RESEARCH.md)
    4. Checks if the tag already exists — exits with an error if `v{version}` already exists in git tags
    5. Creates a git tag `v{version}`
    6. Pushes main branch to origin
    7. Pushes the tag to origin
    8. Prints "Done. v{version} is live."

    Use `set -e` at the top. Use `SCRIPT_DIR` pattern for portable path resolution:
    ```bash
    SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
    VERSION=$(tr -d '[:space:]' < "$SCRIPT_DIR/../VERSION")
    ```

    Make the file executable: `chmod +x scripts/release.sh`.

    The script assumes the remote is pre-configured — no remote setup logic needed.
  </action>
