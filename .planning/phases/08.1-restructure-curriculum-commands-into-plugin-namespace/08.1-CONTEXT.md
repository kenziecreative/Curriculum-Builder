# Phase 08.1: Restructure Curriculum Commands into Plugin Namespace - Context

**Gathered:** 2026-03-22
**Status:** Ready for planning

<domain>
## Phase Boundary

Restructure the 12 curriculum commands and 2 curriculum-specific agents from their current flat layout in `.claude/commands/` and `.claude/agents/` into a proper Claude Code plugin at `.claude/plugins/curriculum/`. Commands become `/curriculum:intake`, `/curriculum:init`, etc. All internal command references updated to match. Dev tooling (sanity-check, security-scanner, secrets-env-auditor) stays outside the plugin.

</domain>

<decisions>
## Implementation Decisions

### Plugin identity
- Plugin name: `curriculum`
- Manifest location: `.claude/plugins/curriculum/.claude-plugin/plugin.json`
- Commands invoked as `/curriculum:intake`, `/curriculum:init`, `/curriculum:approve`, etc.
- Project-local scope — plugin activated via `.claude/settings.json`, travels with the repo

### What moves into the plugin
- All 12 `knz-*` commands from `.claude/commands/` → `.claude/plugins/curriculum/commands/`
- `knz-validator.md` and `session-generator.md` from `.claude/agents/` → `.claude/plugins/curriculum/agents/`

### What stays outside the plugin
- `sanity-check.md` — dev/project tooling, not curriculum-specific
- `security-scanner.md` — general-purpose security agent
- `secrets-env-auditor.md` — general-purpose secrets agent

### File naming
- Drop the `knz-` prefix on all command files: `knz-intake.md` → `intake.md`, `knz-init.md` → `init.md`, `knz-approve.md` → `approve.md`, etc.
- Agent files can keep their descriptive names or drop prefix — Claude's discretion
- Plugin name (`curriculum`) handles scoping; prefix is redundant

### Internal references
- All `/knz-*` invocation references inside command files must be updated to `/curriculum:*` in this phase
- No dangling old-style references left after migration

### Skills conversion
- Not in this phase — commands stay as flat `.md` files in `commands/`
- Converting to `skills/<name>/SKILL.md` format is a separate follow-up phase

### Claude's Discretion
- Exact `plugin.json` field set (beyond required `name`)
- Whether `.claude/settings.json` activation uses `enabledPlugins` key or another mechanism
- Agent file naming convention inside the plugin

</decisions>
