  <action>
    Create the plugin directory structure and manifest:

    1. Create directory `.claude/plugins/curriculum/.claude-plugin/`
    2. Write `.claude/plugins/curriculum/.claude-plugin/plugin.json` with this exact content:
    ```json
    {
      "name": "curriculum",
      "version": "1.0.0",
      "description": "Curriculum builder plugin — encodes pedagogical doctrine as structural constraints to produce delivery-ready curriculum for adult learners",
      "author": {
        "name": "Kelsey Ruger",
        "email": "kelsey@kenziecreative.com"
      }
    }
    ```

    CRITICAL structural rule: `commands/` and `agents/` go at the plugin root (`.claude/plugins/curriculum/`), NOT inside `.claude-plugin/`. Only `plugin.json` goes inside `.claude-plugin/`.

    3. Update `.claude/settings.json` to add plugin activation. The current settings.json has only a `hooks` key. Merge in `extraKnownMarketplaces` and `enabledPlugins` at the top level:
    ```json
    {
      "extraKnownMarketplaces": {
        "curriculum-local": {
          "source": {
            "source": "directory",
            "path": ".claude/plugins/curriculum"
          }
        }
      },
      "enabledPlugins": {
        "curriculum@curriculum-local": true
      },
      "hooks": {
        ... existing hooks content unchanged ...
      }
    }
    ```

    IMPORTANT: `enabledPlugins` MUST be in `settings.json` (not `settings.local.json`) due to a known Claude Code merge bug (GitHub issues #27247, #25086) — if it's only in `settings.local.json` it is silently ignored.
  </action>
  <verify>
