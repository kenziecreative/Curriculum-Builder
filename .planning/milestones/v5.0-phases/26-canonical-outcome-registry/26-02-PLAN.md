<objective>
Add stale upstream detection to all downstream generation stages and extend /curriculum:revise to flag downstream stages when registry data changes mid-pipeline. This ensures that when outcomes or other canonical data is modified, downstream stages know they are working against stale inputs and the user is informed before generation proceeds.

Purpose: CANON-03 requires that outcome changes propagate from the registry, not from individual file edits. Stale detection is the mechanism that enforces this -- stages refuse to silently generate against outdated canonical data.
Output: 7 modified SKILL.md files with stale detection and propagation logic.

```markdown
### 5a-ii. Flag downstream stages as potentially stale

After updating the registry, identify which downstream stages have already completed and are now potentially stale. Read the Stage Progress table from `workspace/{project}/STATE.md`.

For each registry section that was modified in Step 5a, identify stages that depend on it:

| Modified Section | Downstream Stages |
|---|---|
| outcome_wording | Stages 3 (assessments), 4 (modules), 5 (sessions), 6 (metaskills), 7 (transfer), 8 (marketing) |
| assessment_criteria | Stages 4 (modules), 5 (sessions) |
| time_allocations | Stage 5 (sessions) |
| learner_profile | Stages 3-8 (all downstream) |

For each downstream stage that has status `complete` in STATE.md AND depends on a modified registry section:
- The existing Step 5b already handles this by identifying files for regeneration. No STATE.md status change is needed here -- the stale detection added in Plan 01 Task 1 will catch it if the user runs the stage again manually.

However, show the user a summary of what is now stale:

> The registry has been updated. These completed stages may now be working from outdated data:
> - {Stage name} (depends on {modified section})
> - ...
>
> Step 5b will identify which specific files need regeneration.

This summary appears before Step 5b's file identification, giving the user context for why files need updating.
```
