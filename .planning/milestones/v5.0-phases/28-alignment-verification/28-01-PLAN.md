Create `.claude/reference/alignment-check-reference.md` as a reference document that generation stage commands load before running their alignment check. Follow the same pattern as `audit-trail-format.md` — a canonical reference loaded by all stages.

The document must define these sections:

**1. Purpose and Scope**
- Alignment check verifies that generated output faithfully uses source material
- Runs for all 7 generation stages (outcomes through marketing) as a draft audit check
- Source material includes: files in workspace/source-material/, domain-research-findings.md (grounding doc from Phase 27)
- If no source material exists AND no grounding doc exists, alignment check is skipped for that stage (nothing to align against)

**2. What Gets Checked (grounding-required areas)**
Define which output areas MUST be grounded in source material when source material exists:
- Outcomes: program outcomes, module outcomes, session outcomes, enduring understandings, essential questions
- Assessments: assessment criteria, performance indicators, rubric standards
- Modules: module descriptions, key concepts, prerequisite rationale
- Sessions: key concept explanations, pre-work content rationale
- Metaskills: thinking skill activation rationale, domain-specific examples
- Transfer: real-work application scenarios, success indicators
- Marketing: learning promises (traceability to outcome IDs), program description claims

Define which areas are expected to be agent-generated (NOT checked):
- Session activities, discussion prompts, time blocks, facilitator guide logistics
- Slide outline structure
- Assessment format/structure (rubric layout, submission instructions)
- Marketing structural framework (PAS/DOS choice), headline formulas, VOC language adaptation

**3. Distortion Detection Rules**

Define three distortion types with detection patterns and examples:

**Qualifier Stripping:**
- Detection: source text contains hedging language (may, might, in some contexts, research suggests, tends to, often, can, sometimes, for many, in certain cases) but output text removes or replaces it with certainty language (always, never, all, every, studies prove, research confirms, will definitely, guarantees)
- Example: Source says "Research suggests peer feedback tends to improve retention" -> Output says "Peer feedback improves retention" -> FLAGGED: qualifier "suggests" and "tends to" stripped
- Severity: BLOCKING — blocks draft promotion

**Range Narrowing:**
- Detection: source text provides a range (numerical or qualitative) but output narrows to a single value or endpoint
- Example: Source says "Programs of 3-6 months show strongest results" -> Output says "A 4-month program shows strongest results" -> FLAGGED: range "3-6 months" narrowed to "4 months"
- Example: Source says "Learners at intermediate to advanced levels" -> Output says "Advanced learners" -> FLAGGED: range narrowed
- Severity: BLOCKING — blocks draft promotion

**Over-Claiming Grounding:**
- Detection: audit trail's "Grounded In" section claims a source backs a content section, but the source file does not actually contain the claimed finding
- How to verify: for each "Grounded In" entry, check that the referenced source file contains text that substantively matches the claimed quote or finding
- Severity: BLOCKING — same severity as distortion. Falsely claiming evidence backing is an integrity issue

**4. Assumed Content Detection**
- After checking grounding-required areas against source material, any content in those areas that has NO source material backing is flagged as "assumed"
- Flagged in the alignment report and audit trail only — never in the curriculum output files
- Not a blocking issue on its own — but surfaced so SME can see what came from evidence vs agent knowledge
- Per CONTEXT.md: structural content (activities, prompts, time blocks) is expected agent-generated and is NOT flagged

**5. Marketing Exception**
- Marketing does NOT get verbatim alignment checks (marketing's job is transformation)
- Marketing DOES get:
  - Traceability check: every learning promise must link to an outcome ID that exists in curriculum-registry.json outcome_wording
  - Distortion checks: marketing cannot claim outcomes the evidence does not support (qualifier stripping and range narrowing still apply to factual claims about the program)
  - Over-claiming check: if audit trail says a marketing claim is "Grounded In" a source, verify the source actually supports that claim

**6. Alignment Report Format**
Define the report shown during draft audit (not written to any file — displayed in conversation only):

```
### Alignment Check Results

**Status:** PASS / FAIL ({N} issues found)

{For each issue:}
**Issue {N}: {type — Qualifier Stripped / Range Narrowed / Over-Claimed Grounding / Unsupported Claim}**
- Source says: "{verbatim quote from source material}"
- Output says: "{verbatim text from generated draft}"
- Location: {file path and section name}
- Severity: Blocking / Warning

{If assumed content found:}
**Assumed Content (not from source material):**
- {section name}: {brief description of what was generated without source backing}

{If "Read but Not Referenced" items exist:}
**Read but Not Referenced:**
- {source file}: loaded but not incorporated into this stage's output
```

Plain language throughout — no jargon, no check IDs.

**7. Retry Constraint Format**
When alignment check fails and triggers retry, the specific issues are injected as constraints:

```
The previous draft had these alignment issues:
- {Issue type}: Source says "{source quote}", but your output said "{output text}". Preserve the qualifier / range / do not claim this grounding.
- {repeat for each issue}

Regenerate with these specific corrections. All other content can remain the same.
```

**8. Integration Pattern**
- Alignment check runs AFTER all existing draft audit checks pass (vocabulary, registry, schema, etc.)
- If alignment check fails, it follows the same 3-attempt retry with cumulative constraints pattern
- Alignment issues are never auto-fixable — the stage must re-generate
- On escalation (3 failed attempts), alignment issues are listed in the escalation message with the same plain-language format as other check failures
