  <action>
Read each of the 7 generation stage SKILL.md files fully before modifying. Then add trail-writing logic to each.

**The pattern is the same for all 7 stages -- adapt per stage:**

**Where to insert:** In the approval/file-writing branch of each stage. Look for the section where output files are written to workspace (e.g., "On Approval", or draft promotion after audit passes). Add the trail write step AFTER generation output is written but BEFORE the STATE.md update and "next step" message.

**What to add to each stage:**

```
N. Update audit trail:

   Load `.claude/reference/audit-trail-format.md` for the canonical format.

   Read `workspace/{project}/audit-trail.md`. If Stage {N}'s section already exists (re-generation), replace it. Otherwise append.

   Write Stage {N} section following the format reference:

   **Grounded In:** For each major output section produced by this stage, list:
   - The output section name (e.g., "Module 2: Data Analysis", "Session 3 pre-work activity")
   - Which source material file the content drew from
   - The specific claim, finding, or quote that grounded the content

   **Agent-Generated:** List content produced from the agent's own knowledge:
   - Each bullet = one piece of generated content not traceable to source material
   - Be specific (e.g., "Session sequencing rationale", "Assessment rubric criteria") not vague

   **Read but Not Referenced:** List source material files that were loaded for this stage but whose content was not incorporated into the output.

   If no source materials exist: use the no-source-material variant (all content listed under Agent-Generated, no Grounded In or Read but Not Referenced sections).

   Update the Build Summary block at the top of the trail:
   - Add this stage to "Stages completed" list
   - Recalculate grounding percentage

   Do this silently -- no announcement to the user.
```

**Stage-specific details:**

- **Outcomes (Stage 2):** Output sections = program outcomes, module outcomes, session outcomes. Also note: enduring understandings and essential questions as separate items.
- **Assessments (Stage 3):** Output sections = each assessment (formative and summative). The PIPE-05 gate SME confirmation is handled in Task 2 (assessments has an inline gate).
- **Modules (Stage 4):** Output sections = each module specification. Trail write goes after draft audit passes and drafts are promoted.
- **Sessions (Stage 5):** Output sections = each session plan. Trail write goes after draft audit passes and drafts are promoted.
- **Metaskills (Stage 6):** Output sections = metaskill integration plan sections.
- **Transfer (Stage 7):** Output sections = pre-program, in-program, post-program transfer activities.
- **Marketing (Stage 8):** Output sections = each marketing claim / copy section.

**For stages with draft-then-audit (modules, sessions, metaskills, transfer, marketing):**
The trail section write happens AFTER the draft audit passes and files are promoted from `_drafts/` to the stage directory -- because the trail should reflect the final promoted content, not the pre-audit draft.

**Important:** Each stage must also load `.claude/reference/audit-trail-format.md` as a prerequisite read (add to the "Load and generate" or equivalent section alongside other reference doc loads). This ensures the format is available before the trail write step.
