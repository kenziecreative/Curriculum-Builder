<task type="auto">
  <name>Task 1: Create audit trail format reference document</name>
  <files>.claude/reference/audit-trail-format.md</files>
  <action>
Create `.claude/reference/audit-trail-format.md` -- the canonical reference that all stage commands will load before writing their trail section.

The document must define:

**1. File location and lifecycle:**
- Trail lives at `workspace/{project}/audit-trail.md`
- Intake initializes the file; each subsequent stage appends its section
- On re-generation, the stage's section is replaced (not appended) -- trail reflects current state
- Trail is an internal build artifact -- not included in HTML delivery

**2. Top-level summary block (written by intake, updated by each stage):**
```
# Audit Trail: {program_name}
Generated: {date}

## Build Summary
- **Stages completed:** {list}
- **Source materials:** {count} files loaded
- **Grounding:** {percentage} of content grounded in source material
- **SME checkpoints:** {count} confirmations recorded
- **Modifications:** {count} changes at review gates
- **Revisions:** {count} post-delivery revisions
```

**3. Stage section template (each generation stage writes one):**
```
## Stage {N}: {Stage Name}
Generated: {ISO timestamp}

### Grounded In
{For each major output section (e.g., "Module 2 outcomes", "Session 3 pre-work"):}
- **{section name}**: {source file} -- "{relevant quote or finding}"

### Agent-Generated
- {bullet list of content created from agent's own knowledge, not source material}

### Read but Not Referenced
- {source files that were loaded but not used by this stage -- surfaces ignored evidence for Phase 28}

### SME Confirmation
{Only present if this stage has a human gate}
- **Confirmed:** {ISO timestamp}
- **Decision:** {what was confirmed -- e.g., "Approved outcomes as presented"}
- **Modifications:** {if any -- before/after values and SME's reason if stated}
```

**4. No-source-material variant:**
When no source materials exist (from-scratch build), the header includes: "No source materials available -- all content is agent-generated" and stage sections use a simplified format listing only what was generated (no Grounded In or Read but Not Referenced sections).

**5. Revision entry template (used by /curriculum:revise):**
```
## Revision: {date}
- **Feedback:** {what the SME observed}
- **Changes:** {what was changed}
- **Affected stages:** {which stages were updated}
```

**6. Format rules:**
- Chronological order -- stages appear in pipeline order (intake first, marketing last)
- Each stage section is self-contained -- readable without context from other sections
- Use `---` horizontal rules between stage sections
- Grounding percentage in summary = (grounded sections / total sections) as rough estimate
