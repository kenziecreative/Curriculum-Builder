This extends the existing prerequisite pattern -- currently commands check predecessor STATUS only; now they also check that the actual DATA they need exists and is well-formed.

**Pattern for each command (insert as a new numbered prerequisite step):**

```markdown
### N. Input Validation

Read `workspace/{project}/curriculum-registry.json`. Verify the following fields exist and are non-empty:

{stage-specific field list}

If any field is missing or empty, stop and report:

> Cannot start {stage name} -- {specific field description} is missing from the registry. Run {predecessor command} to generate it.

```

**Per-command validation checks:**

- **modules.md** (Stage 4): Validate `outcome_wording` has module_outcomes with bloom_level fields. Validate `assessment_criteria` has at least one assessment with mapped_outcomes. Validate `learner_profile.data.contact_hours` exists.

- **marketing.md** (Stage 8): Validate `learner_profile.data` exists with: target_audience, transfer_context. Validate `outcome_wording.program_outcomes` exists. Validate `assessment_criteria` exists.

Modify the file-write sections of modules.md, sessions.md, and metaskills.md so that generated content writes to a `_drafts` subdirectory first, then runs four audit checks before promoting to the final stage directory.

**Draft write location per the user decision:**
- modules.md: `workspace/{project}/04-modules/_drafts/` (or `03-modules/_drafts/` for legacy scheme)
- sessions.md: `workspace/{project}/05-sessions/_drafts/` (or `04-sessions/_drafts/` for legacy scheme)
- metaskills.md: `workspace/{project}/06-metaskills/_drafts/` (or `05-metaskills/_drafts/` for legacy scheme)

**Changes to each command:**

1. **Redirect file writes to _drafts:** In the section where each command writes its output files, change the output directory from the stage directory to `{stage-dir}/_drafts/`. For sessions.md, the Task subagent dispatch instructions must specify `_drafts/` as the output directory.

2. **Add a "Draft Audit" section** after file writes complete but before the completion summary. This section runs four checks:

```markdown
## Draft Audit

Run these four checks against the files in `{stage-dir}/_drafts/`. All four must pass before promotion.

### Check 1: File Completeness
Verify all expected files exist in _drafts with non-zero content:
{stage-specific file list -- e.g., for modules: one module-spec.md per module directory}

### Check 2: Registry Consistency
Read `workspace/{project}/curriculum-registry.json`. For each outcome ID, module ID, or assessment reference in the draft files, verify it exists in the registry. Flag any ID that appears in drafts but not in the registry.

### Check 3: Vocabulary Scan
Read `.claude/reference/curriculum-voice.md` never-say table. Scan all draft files for prohibited terms. List any violations found with file path and line content.

### Check 4: Schema Compliance
Read the stage schema (`.claude/reference/schemas/stage-{NN}-{name}.md`). Verify all required fields are present in draft files. Flag any missing required fields.

### Audit Result

If all four checks pass: promote files from `_drafts/` to the parent stage directory (move, not copy). Delete the `_drafts/` directory after successful promotion.

If any check fails:
1. Attempt auto-fix for simple failures:
   - Vocabulary violations: substitute with the plain-language replacement from curriculum-voice.md
   - Missing fields that have obvious default values: fill in from registry data
2. Re-run the failing check(s) after auto-fix.
3. If still failing after auto-fix: stop and report the specific failures to the user. Do not promote. Do not mark the stage complete. The user must address the issues and re-run the stage command.

Report format for failures:
> Draft audit found {N} issue(s) that could not be auto-fixed:
> - {file}: {specific problem}
>
> Fix these issues and run {command} again.
```

**Directory scheme awareness:**
